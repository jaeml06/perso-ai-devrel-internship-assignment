# Feature Specification: AI 더빙 코어 기능 (ElevenLabs TTS)

**Feature Branch**: `feat/#3-ai-dubbing-core`
**Created**: 2026-03-09
**Status**: Draft
**GitHub Issue**: #3
**Input**: PR2 - ElevenLabs API를 활용한 텍스트→음성 변환(TTS) 더빙 웹 서비스 핵심 기능 구현. API 키 생성 과정 포함.

## User Scenarios & Testing _(mandatory)_

### User Story 1 - ElevenLabs API 키 준비 (Priority: P1)

개발자(서비스 운영자)가 ElevenLabs 대시보드에서 API 키를 생성하고, 프로젝트 환경 변수에 설정하여 TTS 기능이 동작할 수 있는 기반을 마련한다.

**Why this priority**: API 키 없이는 어떤 TTS 기능도 동작하지 않으므로 가장 선행되어야 하는 작업이다.

**Independent Test**: 환경 변수가 올바르게 설정되었는지 확인하고, ElevenLabs API에 간단한 요청(예: voices 목록 조회)을 보내 유효한 응답을 받는다.

**Acceptance Scenarios**:

1. **Given** ElevenLabs 계정이 있는 개발자, **When** 대시보드 > API 키 메뉴에서 새 키를 생성하고 환경 변수(`ELEVENLABS_API_KEY`)에 설정, **Then** 서버 측에서 API 키를 통해 ElevenLabs API 호출이 정상 동작한다.
2. **Given** API 키가 환경 변수에 설정되지 않은 상태, **When** TTS 기능을 호출, **Then** 시스템은 "API 키가 설정되지 않았습니다"라는 명확한 에러 메시지를 반환한다.

---

### User Story 2 - 텍스트를 음성으로 변환 (Priority: P1)

인가된 사용자가 텍스트를 입력하고 음성(보이스)을 선택한 뒤 "더빙 생성" 버튼을 누르면, 시스템이 입력된 텍스트를 선택된 음성으로 변환하여 오디오를 생성한다.

**Why this priority**: 서비스의 핵심 가치인 "텍스트 → 음성 변환" 기능이므로 최우선이다.

**Independent Test**: 대시보드에서 텍스트를 입력하고 음성을 선택하여 더빙을 요청하면 오디오가 생성되어 재생 가능한 상태가 된다.

**Acceptance Scenarios**:

1. **Given** 인가된 사용자가 대시보드에 접근한 상태, **When** 텍스트 입력란에 텍스트를 입력하고 음성을 선택한 뒤 "더빙 생성"을 클릭, **Then** 시스템이 로딩 상태를 표시한 뒤 생성된 오디오를 재생할 수 있는 플레이어를 표시한다.
2. **Given** 텍스트 입력란이 비어 있는 상태, **When** "더빙 생성"을 클릭, **Then** 시스템이 "텍스트를 입력해주세요"라는 유효성 검증 메시지를 표시한다.
3. **Given** 사용자가 텍스트를 입력한 상태, **When** 음성을 선택하지 않고 "더빙 생성"을 클릭, **Then** 시스템이 "음성을 선택해주세요"라는 유효성 검증 메시지를 표시한다.

---

### User Story 3 - 사용 가능한 음성 목록 조회 (Priority: P1)

사용자가 더빙 대시보드에 접근하면, 성별과 나이대로 분류된 4~5개의 대표 음성을 드롭다운으로 제공하고, 선택 시 미리듣기 버튼으로 음성 샘플을 재생할 수 있다.

**Why this priority**: 음성 선택은 TTS 변환의 필수 입력값이므로 핵심 기능과 동급이다.

**Independent Test**: 대시보드 로드 시 음성 드롭다운에 남성/여성, 나이대별로 구분된 음성이 표시되고, 선택 시 미리듣기 버튼으로 해당 음성의 샘플을 재생할 수 있다.

**Acceptance Scenarios**:

1. **Given** 사용자가 대시보드에 접근, **When** 페이지가 로드됨, **Then** 드롭다운에 성별·나이대로 구분된 6개 대표 음성 목록이 표시된다.
2. **Given** 드롭다운에서 음성을 선택한 상태, **When** 미리듣기 버튼을 클릭, **Then** 해당 음성의 샘플 오디오가 재생된다.
3. **Given** 음성 목록 조회 실패, **When** API 에러 발생, **Then** 사용자에게 "음성 목록을 불러올 수 없습니다" 메시지와 재시도 버튼이 표시된다.

---

### User Story 4 - 생성된 오디오 재생 및 다운로드 (Priority: P2)

사용자가 생성된 오디오를 웹 브라우저에서 바로 재생하거나, 파일로 다운로드할 수 있다.

**Why this priority**: 생성된 오디오의 활용(재생/다운로드)은 핵심 변환 기능 다음으로 중요한 사용자 경험이다.

**Independent Test**: 오디오 생성 후 플레이어에서 재생/일시정지/탐색이 가능하고, 다운로드 버튼으로 파일을 저장할 수 있다.

**Acceptance Scenarios**:

1. **Given** 오디오가 생성된 상태, **When** 재생 버튼을 클릭, **Then** 오디오가 재생되고 진행 바가 업데이트된다.
2. **Given** 오디오가 재생 중인 상태, **When** 일시정지 버튼을 클릭, **Then** 재생이 일시정지된다.
3. **Given** 오디오가 생성된 상태, **When** 다운로드 버튼을 클릭, **Then** 오디오 파일이 사용자 기기에 다운로드된다.

---

### Edge Cases

- 사용자가 매우 긴 텍스트(예: 5,000자 이상)를 입력하면 어떻게 처리하는가? → ElevenLabs 무료 플랜 제한에 따라 텍스트 길이를 제한하고 안내 메시지를 표시한다.
- API 요청 중 네트워크 오류가 발생하면? → 사용자에게 에러 메시지를 표시하고 재시도 옵션을 제공한다.
- ElevenLabs API 할당량(무료 플랜 크레딧)이 소진되면? → "크레딧이 부족합니다" 메시지를 표시하고 변환을 차단한다.
- 동일 사용자가 짧은 시간 내에 연속으로 여러 요청을 보내면? → 이전 요청이 완료될 때까지 버튼을 비활성화하여 중복 요청을 방지한다.
- API 키가 유효하지 않거나 만료된 경우? → 서버에서 401 응답을 감지하고 관리자에게 알림 또는 사용자에게 "서비스 일시 중단" 메시지를 표시한다.

## Requirements _(mandatory)_

### Functional Requirements

- **FR-001**: 시스템은 서버 측에서 ElevenLabs TTS API와 통신하여 텍스트를 음성으로 변환해야 한다 (API 키가 클라이언트에 노출되지 않아야 함).
- **FR-002**: 시스템은 성별·나이대로 분류된 6개의 대표 음성을 드롭다운으로 제공하고, 선택 시 미리듣기를 지원해야 한다.
- **FR-003**: 사용자는 텍스트를 입력하고 음성을 선택하여 더빙(TTS 변환)을 요청할 수 있어야 한다.
- **FR-004**: 시스템은 텍스트 입력값에 대한 유효성 검증(빈 텍스트, 최대 길이 초과)을 수행해야 한다.
- **FR-005**: 시스템은 음성 선택 여부에 대한 유효성 검증을 수행해야 한다.
- **FR-006**: 시스템은 TTS 변환 중 로딩 상태를 사용자에게 표시해야 한다.
- **FR-007**: 사용자는 생성된 오디오를 웹 브라우저에서 재생(재생/일시정지/탐색)할 수 있어야 한다.
- **FR-008**: 사용자는 생성된 오디오를 파일로 다운로드할 수 있어야 한다.
- **FR-009**: 시스템은 API 오류(네트워크 에러, 인증 실패, 할당량 초과 등)에 대해 사용자 친화적 에러 메시지를 표시해야 한다.
- **FR-010**: 시스템은 중복 요청을 방지하기 위해 변환 진행 중 요청 버튼을 비활성화해야 한다.
- **FR-011**: 환경 변수(`ELEVENLABS_API_KEY`)가 설정되지 않은 경우 시스템은 명확한 에러를 반환해야 한다.
- **FR-012**: 사용자는 한국어 또는 영어를 선택하여 해당 언어로 텍스트를 음성 변환할 수 있어야 한다.

### Key Entities

- **Voice**: ElevenLabs에서 제공하는 음성 프로필. 고유 ID, 이름, 미리듣기 URL 등의 속성을 가진다.
- **DubbingRequest**: 사용자의 TTS 변환 요청. 입력 텍스트, 선택된 음성 ID, 요청 상태(대기/처리중/완료/실패)를 포함한다.
- **AudioResult**: TTS 변환 결과. 오디오 데이터(바이너리), 포맷, 재생 시간 정보를 포함한다.

## Success Criteria _(mandatory)_

### Measurable Outcomes

- **SC-001**: 사용자가 텍스트 입력부터 오디오 재생까지 30초 이내에 완료할 수 있다 (1,000자 이하 텍스트 기준).
- **SC-002**: TTS 변환 성공률이 95% 이상이다 (API 정상 상태일 때).
- **SC-003**: 사용자가 첫 방문 시 별도의 설명 없이 더빙 기능을 사용할 수 있다 (직관적 UI).
- **SC-004**: 모든 에러 상황에서 사용자에게 5초 이내에 명확한 피드백이 제공된다.

## Assumptions

- ElevenLabs 무료 플랜의 월간 크레딧(10,000자)으로 데모/테스트 용도에 충분하다.
- 인증/인가 시스템(Google OAuth + 화이트리스트)은 PR3에서 별도 구현되므로, 이 PR에서는 인증 없이 구현한다.
- 오디오 포맷은 mp3로 기본 생성한다 (ElevenLabs 기본 출력 포맷).
- 음성 목록은 성별(남성/여성)과 나이대(청년/중년)로 분류된 6개의 premade 대표 음성만 제공한다.
- 텍스트 최대 길이는 ElevenLabs 무료 플랜 제한에 맞춰 5,000자로 설정한다.
- 지원 언어는 한국어와 영어 두 가지이며, 사용자가 선택할 수 있다.
- 더빙 결과는 세션 내에서만 유지되고, 페이지를 떠나면 사라진다 (이력 저장 없음).
