# Feature Specification: AI 더빙 코어 기능 (파일 업로드 → STT → 번역 → TTS)

**Feature Branch**: `feat/#3-ai-dubbing-core`
**Created**: 2026-03-09
**Updated**: 2026-03-10
**Status**: Updated (Requirements Changed)
**GitHub Issue**: #3
**Input**: PR2 (Updated) - 오디오/비디오 파일 업로드 후 원하는 언어로 더빙해주는 전체 파이프라인 구현 (STT → 번역 → TTS). TDD 방식으로 구현. ElevenLabs STT API가 비디오 포맷을 직접 지원하므로 서버 측 별도 변환 불필요.

## Overview

사용자가 오디오 또는 비디오 파일을 업로드하고 타겟 언어를 선택하면, 시스템이 다음 파이프라인을 통해 더빙된 결과물을 생성한다:

1. **음성 추출 & 전사(STT)**: ElevenLabs API로 업로드된 파일에서 음성을 텍스트로 변환
2. **번역**: AI를 통해 전사된 텍스트를 타겟 언어로 번역
3. **음성 합성(TTS)**: ElevenLabs API로 번역된 텍스트를 타겟 언어 음성으로 합성
4. **결과물 제공**: 더빙된 오디오 재생 및 다운로드

## User Scenarios & Testing _(mandatory)_

### User Story 1 - 오디오/비디오 파일 업로드 및 언어 선택 (Priority: P1)

인가된 사용자가 더빙 대시보드에서 오디오 또는 비디오 파일을 업로드하고, 타겟 언어와 TTS 음성을 선택한 뒤 더빙 생성을 요청한다. ElevenLabs STT API가 비디오 포맷을 직접 처리하므로 서버 측 별도 변환 없이 파일을 그대로 전송한다.

**Why this priority**: 더빙 서비스의 시작점이며, 유효한 파일과 언어 선택 없이는 이후 파이프라인이 진행될 수 없다.

**Independent Test**: 파일 업로드 UI에서 지원 포맷(mp3, wav, mp4 등)의 파일을 업로드하고, 타겟 언어와 TTS 음성을 선택하면 "더빙 시작" 버튼이 활성화된다.

**Acceptance Scenarios**:

1. **Given** 인가된 사용자가 대시보드에 접근, **When** 지원 포맷(mp3, wav, mp4, mov, ogg, flac 등)의 파일을 업로드하고 타겟 언어와 음성을 선택, **Then** "더빙 생성" 버튼이 활성화된다.
2. **Given** 지원하지 않는 파일 포맷(예: pdf, txt)을 업로드, **When** 파일 선택 시, **Then** 시스템이 "지원하지 않는 파일 형식입니다" 메시지를 표시하고 업로드를 거부한다.
3. **Given** 파일이 업로드되지 않은 상태, **When** "더빙 생성"을 클릭, **Then** 시스템이 "파일을 업로드해주세요"라는 유효성 검증 메시지를 표시한다.
4. **Given** 타겟 언어가 선택되지 않은 상태, **When** "더빙 생성"을 클릭, **Then** 시스템이 "타겟 언어를 선택해주세요"라는 유효성 검증 메시지를 표시한다.
5. **Given** 음성이 선택되지 않은 상태, **When** "더빙 생성"을 클릭, **Then** 시스템이 "음성을 선택해주세요"라는 유효성 검증 메시지를 표시한다.

### User Story 2 - 음성 전사(STT) (Priority: P1)

시스템이 업로드된 파일에서 ElevenLabs STT API를 통해 음성을 텍스트로 전사한다.

**Why this priority**: 파이프라인의 첫 번째 단계이며, 전사 실패 시 이후 번역/합성이 불가능하다.

**Independent Test**: 알려진 내용의 오디오 파일을 업로드하면 시스템이 정확한 전사 텍스트를 반환한다.

**Acceptance Scenarios**:

1. **Given** 사용자가 유효한 오디오 파일을 업로드하고 더빙을 요청, **When** STT 처리 진행 중, **Then** 시스템이 "음성을 텍스트로 변환 중..." 진행 상태를 표시한다.
2. **Given** 오디오 파일의 STT 처리 완료, **When** 전사 성공, **Then** 원본 언어의 전사 텍스트가 시스템 내부에 저장되어 다음 단계(번역)로 전달된다.
3. **Given** STT API 오류 발생, **When** 전사 실패, **Then** 시스템이 "음성 인식에 실패했습니다. 다시 시도해주세요" 메시지와 재시도 옵션을 표시한다.

---

### User Story 3 - 텍스트 번역 (Priority: P1)

시스템이 전사된 텍스트를 사용자가 선택한 타겟 언어로 번역한다.

**Why this priority**: 번역은 파이프라인의 핵심 중간 단계로, 실패 시 더빙된 음성을 생성할 수 없다.

**Independent Test**: 영어 전사 텍스트를 한국어로 번역 요청 시 자연스러운 한국어 번역 결과가 반환된다.

**Acceptance Scenarios**:

1. **Given** 전사 텍스트가 준비된 상태, **When** 번역 처리 진행 중, **Then** 시스템이 "텍스트를 번역 중..." 진행 상태를 표시한다.
2. **Given** 번역 완료, **When** 번역 성공, **Then** 타겟 언어로 번역된 텍스트가 TTS 단계로 전달된다.
3. **Given** 번역 API 오류 발생, **When** 번역 실패, **Then** 시스템이 "번역에 실패했습니다. 다시 시도해주세요" 메시지와 재시도 옵션을 표시한다.

---

### User Story 4 - 음성 합성(TTS) (Priority: P1)

시스템이 번역된 텍스트를 ElevenLabs TTS API를 통해 타겟 언어 음성으로 합성한다.

**Why this priority**: 파이프라인의 마지막 처리 단계이며, 최종 더빙 결과물을 생성한다.

**Independent Test**: 번역된 한국어 텍스트로 TTS 합성 요청 시 한국어 음성의 오디오 파일이 생성된다.

**Acceptance Scenarios**:

1. **Given** 번역 텍스트가 준비된 상태, **When** TTS 처리 진행 중, **Then** 시스템이 "음성을 합성 중..." 진행 상태를 표시한다.
2. **Given** TTS 완료, **When** 합성 성공, **Then** 타겟 언어로 더빙된 오디오가 생성되어 재생/다운로드 가능한 상태가 된다.
3. **Given** TTS API 오류 발생, **When** 합성 실패, **Then** 시스템이 "음성 합성에 실패했습니다. 다시 시도해주세요" 메시지와 재시도 옵션을 표시한다.

---

### User Story 5 - 더빙된 결과물 재생 및 다운로드 (Priority: P2)

사용자가 생성된 더빙 오디오를 웹 브라우저에서 바로 재생하거나 파일로 다운로드할 수 있다.

**Why this priority**: 생성된 더빙 결과물의 최종 전달 단계이다.

**Independent Test**: 더빙 생성 완료 후 플레이어에서 재생/일시정지/탐색이 가능하고, 다운로드 버튼으로 파일을 저장할 수 있다.

**Acceptance Scenarios**:

1. **Given** 더빙 오디오가 생성된 상태, **When** 재생 버튼을 클릭, **Then** 오디오가 재생되고 진행 바가 업데이트된다.
2. **Given** 오디오가 재생 중인 상태, **When** 일시정지 버튼을 클릭, **Then** 재생이 일시정지된다.
3. **Given** 더빙 오디오가 생성된 상태, **When** 다운로드 버튼을 클릭, **Then** 더빙된 오디오 파일이 사용자 기기에 다운로드된다.

---

### Edge Cases

- **대용량 파일 업로드**: 파일 크기가 제한(예: 25MB)을 초과하면 업로드 전에 차단하고 안내 메시지를 표시한다.
- **무음 파일**: 전사 텍스트가 비어 있거나 인식 불가 경우 "음성을 감지하지 못했습니다" 메시지를 표시한다.
- **네트워크 오류**: 파이프라인 각 단계에서 네트워크 오류 발생 시 해당 단계 재시도 옵션을 제공한다.
- **API 할당량 초과**: ElevenLabs 무료 플랜 크레딧 소진 시 "크레딧이 부족합니다" 메시지를 표시하고 변환을 차단한다.
- **동일 언어 선택**: 원본 언어와 타겟 언어가 동일한 경우 번역을 건너뛰고 TTS 단계로 진행한다 (또는 사용자에게 경고).
- **중복 요청 방지**: 파이프라인 진행 중에는 버튼을 비활성화하여 중복 요청을 방지한다.
- **API 키 미설정**: 환경 변수 미설정 시 서버에서 명확한 에러 메시지를 반환한다.

## Requirements _(mandatory)_

### Functional Requirements

- **FR-001**: 시스템은 오디오(mp3, wav, ogg, flac, m4a) 및 비디오(mp4, mov, webm) 파일 업로드를 지원해야 한다. ElevenLabs STT API가 해당 포맷을 직접 처리한다.
- **FR-002**: 시스템은 업로드된 파일의 크기를 검증하고 지원하지 않는 포맷은 거부해야 한다.
- **FR-003**: 시스템은 ElevenLabs STT API를 통해 업로드된 파일에서 음성을 텍스트로 전사해야 한다.
- **FR-004**: 시스템은 AI 번역 서비스를 통해 전사 텍스트를 사용자가 선택한 타겟 언어로 번역해야 한다.
- **FR-005**: 시스템은 ElevenLabs TTS API를 통해 번역된 텍스트를 사용자가 선택한 음성으로 합성해야 한다.
- **FR-005a**: 시스템은 타겟 언어별로 선택 가능한 음성 목록(드롭다운)을 제공해야 하며, 성별·나이대로 구분된 대표 음성을 포함해야 한다.
- **FR-006**: 시스템은 STT → 번역 → TTS 각 파이프라인 단계별 진행 상태(로딩 표시)를 사용자에게 표시해야 한다.
- **FR-007**: 사용자는 타겟 언어를 선택할 수 있어야 하며, 한국어·영어 2개 언어를 지원한다.
- **FR-008**: 사용자는 생성된 더빙 오디오를 웹 브라우저에서 재생(재생/일시정지/탐색)할 수 있어야 한다.
- **FR-009**: 사용자는 생성된 더빙 오디오를 파일로 다운로드할 수 있어야 한다.
- **FR-010**: 시스템은 파이프라인 각 단계의 API 오류에 대해 사용자 친화적 에러 메시지와 재시도 옵션을 제공해야 한다.
- **FR-011**: 시스템은 파이프라인 진행 중 중복 요청을 방지하기 위해 입력 UI를 비활성화해야 한다.
- **FR-012**: 모든 API 통신은 서버 사이드에서 처리되어야 하며 API 키가 클라이언트에 노출되지 않아야 한다.
- **FR-013**: 환경 변수(`ELEVENLABS_API_KEY`, `GEMINI_API_KEY`)가 설정되지 않은 경우 시스템은 해당 엔드포인트에서 명확한 에러를 반환해야 한다.

### Key Entities

- **DubbingJob**: 사용자의 더빙 요청. 업로드 파일 정보, 타겟 언어, 파이프라인 상태(대기/전사중/번역중/합성중/완료/실패)를 포함한다.
- **TranscriptionResult**: STT 결과. 원본 텍스트, 감지된 언어, 신뢰도를 포함한다.
- **TranslationResult**: 번역 결과. 번역된 텍스트, 소스 언어, 타겟 언어를 포함한다.
- **AudioResult**: TTS 합성 결과. 오디오 데이터(바이너리), 포맷, 재생 시간 정보를 포함한다.

## Success Criteria _(mandatory)_

### Measurable Outcomes

- **SC-001**: 사용자가 파일 업로드부터 더빙 오디오 재생까지 전체 파이프라인을 완료할 수 있다 (3분 이하 분량 파일 기준).
- **SC-002**: 파이프라인의 각 단계(STT/번역/TTS)가 독립적으로 테스트 가능하다 (TDD).
- **SC-003**: 지원 오디오(mp3, wav, ogg, flac, m4a) 및 비디오(mp4, mov, webm) 포맷 파일에 대해 더빙 성공률이 95% 이상이다 (API 정상 상태 기준).
- **SC-004**: 모든 에러 상황에서 사용자에게 5초 이내에 명확한 피드백이 제공된다.
- **SC-005**: 사용자가 첫 방문 시 별도의 설명 없이 더빙 기능을 사용할 수 있다 (직관적 UI).

## Assumptions

- ElevenLabs 무료 플랜의 API로 STT 및 TTS 기능 모두 사용 가능하다.
- 번역 AI는 Google Gemini API(무료 플랜 - `gemini-3.1-flash-lite-preview` 모델)를 사용한다. 해당 모델은 무료 플랜에서 사용 가능하며 과제 전형 목적에 적합하다.
- 오디오 출력 포맷은 mp3로 기본 생성한다 (ElevenLabs 기본 출력 포맷).
- 파일 크기 제한은 25MB로 설정한다 (ElevenLabs API 제한 기준).
- 더빙 결과는 세션 내에서만 유지되고, 페이지를 떠나면 사라진다 (이력 저장 없음).
- 인증/인가 시스템(Google OAuth + 화이트리스트)은 PR3에서 별도 구현되므로, 이 PR에서는 인증 없이 구현한다.
- TDD 방식으로 구현: 각 파이프라인 함수(STT, 번역, TTS)에 대한 단위 테스트를 먼저 작성한 뒤 구현한다.
