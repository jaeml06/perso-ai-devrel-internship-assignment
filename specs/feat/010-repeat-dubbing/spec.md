# Feature Specification: 연속 더빙 생성 복구

**Feature Branch**: `feat/#10-repeat-dubbing`  
**Created**: 2026-03-10  
**Status**: Draft  
**GitHub Issue**: #10  
**Input**: User description: "현재 버그가 있는 것 같아. 최초에 더빙을 생성하면 새로고침을 해야만 다시 생성할 수 있는 것이 문제인것 같아."

## 배경 및 목적

현재 사용자는 더빙을 한 번 생성한 뒤 같은 화면에서 다시 더빙을 생성하려고 하면 작업을 이어갈 수 없고, 페이지를 새로고침해야만 다음 생성이 가능하다. 이 문제는 반복 사용 흐름을 끊고, 실험적으로 파일·언어·음성을 바꾸며 결과를 비교하려는 핵심 사용 패턴을 방해한다.

이 기능의 목적은 더빙 생성 완료 후에도 사용자가 같은 세션 안에서 연속으로 더빙을 생성할 수 있게 하고, 이전 작업의 상태가 다음 작업을 막지 않도록 생성 흐름을 안정화하는 것이다.

---

## User Scenarios & Testing _(mandatory)_

### User Story 1 - 새로고침 없이 다음 더빙 생성 (Priority: P1)

사용자는 첫 번째 더빙 생성이 끝난 뒤 페이지를 벗어나지 않고 바로 두 번째 더빙 생성을 시작할 수 있다.

**Why this priority**: 현재 가장 명확한 장애이며, 해결되지 않으면 더빙 기능을 반복적으로 사용하는 기본 흐름이 성립하지 않는다.

**Independent Test**: 한 번 더빙을 완료한 직후 파일, 언어, 음성 중 하나 이상을 바꿔 다시 생성 요청을 보냈을 때 새로고침 없이 새 파이프라인이 시작되면 된다.

**Acceptance Scenarios**:

1. **Given** 사용자가 더빙 생성에 성공한 상태, **When** 같은 화면에서 다시 생성 요청을 보낼 때, **Then** 시스템은 새로고침 없이 새 더빙 작업을 시작해야 한다.
2. **Given** 사용자가 첫 번째 결과를 본 직후, **When** 새 파일을 선택하고 생성 요청을 보낼 때, **Then** 이전 결과와 분리된 새 작업으로 처리되어야 한다.
3. **Given** 사용자가 첫 번째 결과를 본 직후, **When** 언어나 음성만 바꿔 다시 생성 요청을 보낼 때, **Then** 시스템은 변경된 조건으로 새 작업을 시작해야 한다.

---

### User Story 2 - 이전 작업 상태의 안전한 초기화 (Priority: P2)

사용자가 새 더빙 생성을 시작하면 이전 작업의 완료 상태, 재생 결과, 오류 메시지, 진행 상태가 다음 작업을 방해하지 않도록 적절히 초기화된다.

**Why this priority**: 단순히 버튼만 다시 누를 수 있게 하는 것으로는 부족하고, 이전 실행의 잔여 상태가 남으면 잘못된 결과 표시나 중복 오해가 생긴다.

**Independent Test**: 성공 상태에서 결과가 보이는 화면에서 다시 생성 요청을 보냈을 때 이전 결과 표시가 새 작업의 진행 상태와 충돌하지 않는지 확인하면 된다.

**Acceptance Scenarios**:

1. **Given** 직전 더빙이 성공하여 결과가 보이는 상태, **When** 새 생성 요청이 시작될 때, **Then** 이전 결과 표시와 완료 상태는 새 진행 상태에 맞게 정리되어야 한다.
2. **Given** 직전 더빙이 실패한 상태, **When** 사용자가 조건을 수정하고 다시 생성 요청을 보낼 때, **Then** 이전 오류 메시지가 새 작업 시작을 막아서는 안 된다.
3. **Given** 새 생성 요청이 시작된 상태, **When** 사용자가 진행 상황을 확인할 때, **Then** 현재 실행 중인 작업의 단계만 보아야 하며 이전 실행 상태와 혼동되지 않아야 한다.

---

### User Story 3 - 반복 생성 중 예측 가능한 입력 제어 (Priority: P3)

사용자는 생성이 진행 중일 때만 입력이 잠기고, 완료 또는 실패 후에는 다시 조건을 바꿔 새 생성을 시작할 수 있다.

**Why this priority**: 반복 생성 흐름에서 입력 제어 규칙이 명확해야 사용자가 언제 다시 시도할 수 있는지 판단할 수 있다.

**Independent Test**: 진행 중에는 중복 요청이 막히고, 완료 또는 실패 직후에는 입력을 다시 수정한 뒤 곧바로 생성할 수 있으면 된다.

**Acceptance Scenarios**:

1. **Given** 더빙 생성이 진행 중인 상태, **When** 사용자가 다시 생성 버튼을 누르거나 입력을 바꾸려 할 때, **Then** 현재 작업이 끝날 때까지 중복 실행은 방지되어야 한다.
2. **Given** 더빙 생성이 완료된 상태, **When** 사용자가 다시 입력을 수정할 때, **Then** 시스템은 입력 변경과 다음 생성 요청을 허용해야 한다.
3. **Given** 더빙 생성이 실패한 상태, **When** 사용자가 그대로 재시도하거나 일부 입력을 바꿔 재시도할 때, **Then** 시스템은 새 작업으로 다시 시작할 수 있어야 한다.

---

### Edge Cases

- 사용자가 첫 번째 결과를 재생 중인 상태에서 새 생성을 시작하면 이전 결과 재생은 새 작업을 방해하지 않아야 한다.
- 사용자가 동일한 파일과 동일한 옵션으로 연속 생성하더라도 시스템은 이를 새 작업으로 처리해야 한다.
- 실패 직후 재시도한 경우 이전 실패 메시지가 남아 있어도 새 작업 시작과 동시에 현재 작업 기준으로 갱신되어야 한다.
- 새 생성 요청 직전에 입력이 일부 비어 있거나 유효하지 않으면, 기존과 동일하게 유효성 검증 메시지를 보여주고 작업은 시작하지 않아야 한다.
- 진행 중 상태에서만 중복 요청이 차단되어야 하며, 완료 상태 자체가 재생성을 막는 잠금 상태로 남아서는 안 된다.

## Requirements _(mandatory)_

### Functional Requirements

- **FR-001**: 시스템은 더빙 생성이 성공적으로 완료된 이후에도 같은 화면에서 새 더빙 생성을 다시 시작할 수 있어야 한다.
- **FR-002**: 시스템은 더빙 생성이 실패한 이후에도 사용자가 입력을 유지하거나 수정한 뒤 새 작업을 다시 시작할 수 있어야 한다.
- **FR-002a**: 시스템은 실패 상태에서 사용자가 `다시 시도`를 선택하면 현재 유지 중인 입력값으로 즉시 새 더빙 생성을 다시 시작할 수 있어야 한다.
- **FR-003**: 시스템은 새 더빙 생성을 시작할 때 이전 작업의 완료 상태, 오류 상태, 결과 표시를 현재 작업 기준으로 갱신해야 한다.
- **FR-003a**: 시스템은 새 더빙 생성이 시작되는 즉시 직전 성공 작업의 오디오 결과 표시를 숨겨야 한다.
- **FR-004**: 시스템은 더빙 생성이 진행 중일 때만 중복 실행을 방지해야 하며, 완료 상태나 실패 상태에서는 다음 생성 요청을 막아서는 안 된다.
- **FR-005**: 사용자는 새 파일 선택, 언어 변경, 음성 변경 중 하나 이상을 수행한 뒤 새 더빙 생성을 요청할 수 있어야 한다.
- **FR-005a**: 시스템은 더빙 완료 또는 실패 이후에도 파일, 언어, 음성 입력값을 유지하여 사용자가 같은 조건 또는 일부 수정된 조건으로 다음 생성을 바로 시작할 수 있게 해야 한다.
- **FR-006**: 시스템은 동일한 입력 조합으로 연속 생성 요청이 들어와도 각 요청을 별개의 더빙 작업으로 처리해야 한다.
- **FR-007**: 시스템은 새 더빙 생성 요청이 시작되면 현재 실행 중인 작업 기준의 진행 상태만 사용자에게 표시해야 한다.
- **FR-008**: 시스템은 유효하지 않은 입력 상태에서는 기존과 동일한 검증 규칙을 적용하고, 유효성 검증 실패 시 새 작업을 시작하지 않아야 한다.
- **FR-009**: 시스템은 완료 후 재생성 가능 여부를 사용자가 별도 새로고침 없이 1회 조작으로 확인할 수 있게 해야 한다.

### Key Entities _(include if feature involves data)_

- **Dubbing Request**: 사용자가 현재 화면에서 보내는 한 번의 더빙 생성 요청. 선택한 파일, 언어, 음성과 함께 개별 실행 단위를 나타낸다.
- **Generation Session State**: 현재 화면에서 보이는 더빙 생성 상태. 입력값, 진행 상태, 오류 메시지, 결과 표시 여부를 포함한다.
- **Persistent Inputs**: 반복 생성 동안 화면에 유지되는 사용자 선택값. 파일, 타겟 언어, 음성 선택을 포함하며 완료/실패 후에도 다음 시도를 위해 유지된다.
- **Generation Result**: 직전에 성공한 더빙 작업의 결과물. 재생 가능 여부와 다운로드 가능한 결과를 포함하며, 다음 작업 시작 시 표시 상태가 조정된다.

## Success Criteria _(mandatory)_

### Measurable Outcomes

- **SC-001**: 사용자는 페이지 새로고침 없이 연속으로 두 번 이상 더빙 생성을 완료할 수 있다.
- **SC-002**: 완료 직후 다시 생성 요청을 보냈을 때 새 작업이 즉시 시작되며, 사용자가 별도 우회 행동을 하지 않아도 된다.
- **SC-003**: 실패 후 재시도 흐름에서 사용자는 입력 수정 여부와 무관하게 다음 생성 요청을 시작할 수 있다.
- **SC-003a**: 실패 직후 `다시 시도`를 선택하면 별도 입력 수정 없이도 즉시 다음 작업이 시작된다.
- **SC-004**: 반복 생성 시 이전 작업 상태와 현재 작업 상태가 동시에 노출되어 사용자를 혼동시키는 상황이 재현되지 않는다.
- **SC-005**: 진행 중 상태에서만 중복 요청이 차단되고, 완료 또는 실패 상태는 재생성 가능한 상태로 유지된다.

## Assumptions

- 더빙 생성 이력 저장 기능은 없으며, 문제 범위는 현재 페이지 세션 내 반복 생성 가능 여부에 한정한다.
- 반복 생성은 동일 화면 안에서 파일, 언어, 음성을 다시 선택하거나 그대로 유지한 상태에서 다시 요청하는 흐름을 모두 포함한다.
- 사용자는 한 번에 하나의 더빙 작업만 실행하며, 병렬 생성 지원은 이번 범위에 포함하지 않는다.
