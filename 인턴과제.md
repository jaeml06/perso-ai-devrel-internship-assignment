## 과제 개요

이 과제는 **코딩 에이전트(Claude Code, Google Antigravity 등)를 적극 활용**하여 AI 더빙 웹 서비스를 직접 구축하고, 그 결과물을 개발자 커뮤니티에 홍보하는 것을 목표로 합니다.

DevRel의 핵심 역량인 **"직접 만들고 → 공유하는"** 사이클을 실제로 경험하는 과제입니다.

---

## 과제 1. AI 더빙 웹 서비스 구현

### 사전 준비 (과제 시작 전 완료)

아래 서비스에 무료로 가입하세요.

| 서비스     | 용도                                   | 링크                  |
| ---------- | -------------------------------------- | --------------------- |
| GitHub     | 코드 저장 및 배포 트리거               | https://github.com    |
| Vercel     | 웹 서비스 배포 (무료 플랜)             | https://vercel.com    |
| Turso      | 데이터베이스 (무료 플랜)               | https://turso.tech    |
| ElevenLabs | 음성 전사 및 음성 합성 API (무료 플랜) | https://elevenlabs.io |

> GitHub 계정이 없는 경우 신규 개설하세요.
> ElevenLabs 무료 API 키는 가입 후 대시보드에서 발급 가능합니다.

---

### 1-1. 음성 더빙 서비스 구현

**프레임워크:** Next.js

비디오 또는 오디오 파일을 업로드하면 원하는 언어로 더빙해주는 웹 서비스를 구현하세요.

**입력:**

- 오디오 또는 비디오 파일
- 타겟 언어 (더빙할 언어)

**출력:**

- 타겟 언어로 더빙된 오디오 또는 비디오 파일

**처리 흐름:**

1. 업로드된 파일에서 음성 추출 및 전사 — **ElevenLabs API 사용**
2. 전사된 텍스트를 타겟 언어로 번역 — **번역 AI는 자유롭게 선택하여 구현**
3. 번역된 텍스트를 타겟 언어 음성으로 합성 — **ElevenLabs API 사용**
4. 더빙된 결과물 재생 및 다운로드 제공

구체적인 UI 및 지원 언어 범위는 자유롭게 결정하되, 위 입출력 스펙은 반드시 충족해야 합니다.

---

### 1-2. 회원 관리 기능

서비스에 접근할 수 있는 사용자를 제한하는 **화이트리스트 기반 접근 제어**를 구현하세요.

**요구사항:**

- **Google OAuth 로그인** 구현
- 로그인한 사용자의 이메일이 **허용 리스트에 등록된 경우에만** 서비스 이용 가능
- 허용 리스트에 없는 사용자는 접근 차단 (안내 메시지 표시)
- 허용 회원 데이터는 **Turso 데이터베이스**에 저장하여 관리
- 아래 google 계정을 허용 리스트에 반드시 포함:

```
kts123@estsoft.com
```

---

### 1-3. 문서화

프로젝트의 소개 및 주요 내용 문서화는 **코딩 에이전트를 통해** 작성하세요.

**README.md 포함 필수 항목:**

- 서비스 소개 및 주요 기능
- 사용한 기술 스택
- 로컬 실행 방법
- **배포된 서비스 URL**
- 코딩 에이전트 활용 방법 및 노하우

> 문서는 코딩 에이전트가 작성하고 Git에 커밋되도록 처리

---

### 1-4. 배포 자동화

- GitHub에 커밋 & 푸시하면 **Vercel에 자동 배포**되도록 연동 설정
- 배포 완료 후 서비스가 실제로 접근 가능한 상태여야 함

## 과제 2. GitHub 레포지터리 홍보

### 미션

과제 1에서 구축한 GitHub 레포지터리가 **최대한 많은 Star**를 받을 수 있도록 개발자 커뮤니티에 홍보하세요.

### 홍보 방향

- **코딩 에이전트(Claude Code, Google Antigravity 등) 활용**에 초점을 맞춰 홍보
  - 에이전트로 어떻게 개발했는지, 어떤 점이 유용했는지 등 경험을 공유
- 개발자들이 실제로 관심을 가질 만한 채널에 배포
- 예시 채널: X(Twitter), Reddit, Dev.to, Velog, GeekNews, 오픈 카카오톡 개발자 채널 등

### 제출 항목

홍보한 모든 게시물/스레드의 **URL 목록**을 정리하여 함께 제출하세요.

## 평가 기준

### 과제 1

| 항목               | 세부 기준                                                                    |
| ------------------ | ---------------------------------------------------------------------------- |
| 서비스 동작        | 파일 업로드 → 음성 추출 → 더빙 생성 → 결과물 출력 플로우가 실제로 작동하는가 |

| 배포 자동화        | GitHub 푸시 → Vercel 자동 배포가 연동되어 있는가                             |
| 문서화             | README가 코딩 에이전트를 통해 작성되고 필수 항목이 포함되어 있는가           |
| 코딩 에이전트 활용 | 에이전트를 주도적으로 활용한 흔적이 있는가 (커밋 히스토리, README 등)        |
